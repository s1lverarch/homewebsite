Installing on top of Arch Linux

The Silverarch Repository is a specialized package repository for the Silverarch Linux distribution. Add it to any Arch-based system — Arch Linux, ArchARM, or any Arch-based distro.

01
Open your pacman config
Edit your pacman configuration file with your preferred editor.
Bash 
$ sudo nano /etc/pacman.conf
02
Add the Silverarch repository
Paste this at the bottom of your config file.
Bash : pacman.conf 
[silverarch-repo]
SigLevel = Optional TrustAll
Server   = https://raw.githubusercontent.com/
           iyed-one-st/silverarch-repo/main/x86_64/
Once You Complete In Nano Editor Press Ctrl+O To Save And Ctrl+X To Exit
03
Sync & update
Update all your repos and packages.
Bash 
$ sudo pacman -Syy

// origin story

Built for the World.

SilverArch Linux started as a personal project by Silvercore — a Linux distribution engineer from Tunisia who wanted an Arch-based system tuned for real-world performance, not theoretical benchmarks.

What began as a local effort grew into the first Arabic Tunisian Arch-based distribution with a truly international audience — supported by a multilingual community spanning Arabic, English, and Russian speakers.

Silverarch Multi Distro Repositories

A modular container management engine — deploy and manage 50 Linux distribution environments from a single unified CLI.

supported distributions50 environments
sys install → limited to 6 core distros: saur · salp · sfur · subu · sdur · ssol
// install packages
# install in a distro env
$ smdr -S saur neovim
$ smdr -S subu htop
$ smdr -S skali nmap
$ smdr -S stur python
// container lifecycle
# deploy a distro container
$ smdr -adc sfur
# pull & sync image
$ smdr -Sys sdur
# arch arm (mobile)
$ smdr -S sarur vim
smdr -S [distro] [package]  ·  smdr -adc [distro]  ·  smdr -Sys [distro]   Note: ensure distro alias is valid when using -Sys
